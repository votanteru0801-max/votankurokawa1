# 要件定義書 — 黒革の手帳（人事分析LINE Bot）

最終更新: 2026-07-16 / 版: MVP第1段階

## 1. 目的

石橋輝一専用の人事分析LINE Bot「黒革の手帳」を構築する。社員・採用候補者・外部コンサルタント・取引先・講師・パートナー等について、四柱推命・算命学の決定論的な計算結果と、登録済みの人事情報（面談記録・評価・希望キャリア等）を統合し、人事判断を補助する参謀としての回答を行う。占術のみで採用・不採用・昇格・降格・異動・退職勧奨等を自動決定しない。

## 2. 利用者とアクセス制御

- 唯一の正規利用者は石橋輝一。環境変数 `ALLOWED_LINE_USER_ID` に設定されたLINEユーザーIDのみ全機能を利用できる。
- 暗証番号（PIN）認証は行わない。LINEユーザーIDの一致のみで認可する。
- 許可されていないユーザーには「このアカウントでは利用できません。」という定型文のみを返し、社員名・登録人数・機能一覧・エラー詳細など、Botの内部状態を一切開示しない。
- 未許可ユーザーからのメッセージは監査ログに記録するが、人事データへのアクセスやAI呼び出しは一切行わない。

## 3. 対象人物区分

社員 / 採用候補者 / 外部コンサルタント / 取引先 / 講師 / パートナー / その他。

## 4. 主要ユースケース（優先順位順）

1. 社員個人の強み・弱みの理解
2. 大運・年運から見た「動かす時期・待つ時期」の判断
3. 採用候補者の特性理解
4. 面談での伝え方・質問設計
5. 上司・部下の相性確認
6. 向いている役職・役割の検討
7. チーム編成の検討
8. 複数人比較による責任者候補の選定
9. 退職・モチベーション低下の兆候確認
10. 外部コンサルタント・取引先窓口に向く社員の選定

第1段階（MVP）では 1・2・3（分析のみ）・4・9（設計のみ）を実装する。5〜8、10は第2段階、9の自動仮説拡張・海外出生者対応等は第3段階とする（詳細は `docs/phase2-design.md`, `docs/phase3-design.md` を参照）。

## 5. 機能要件

### 5.1 認証
- LINE Webhook署名検証（channel secretによるHMAC-SHA256検証）。
- 許可LINEユーザーIDの照合。
- WebhookイベントIDによる二重処理防止（PostgreSQLに処理済みイベントIDを記録）。

### 5.2 人物登録
- 一括文章入力（例:「採用候補者を登録。山田花子、2003年5月10日、10時30分、福岡県福岡市、女性」）に対応する自然言語パーサー。
- 質問形式（区分→氏名→生年月日→出生時間→出生地→性別→不足確認→最終確認→登録）のマルチターン対話。
- 登録前に内容確認を提示し、「登録する」「修正する」「中止する」に対応する。
- 出生時間が不明な場合は「不明」として登録し、時柱を使用しない分析にフォールバックする。
- 同姓同名が既存データにある場合は、所属・生年月日・人物区分等で本人特定をLINE上で確認する。

### 5.3 命式計算（決定論的エンジン、LLM推測禁止）
- 四柱（年柱・月柱・日柱・時柱）、日干、陰陽、五行、通変星、十二運、蔵干、五行バランス、大運・年運・月運（四柱推命側）。
- 中心星、十大主星、十二大従星、天中殺、守護神、位相法、人体星図、大運（算命学側）。
- 出生時間なしの場合は時柱・時柱依存判定を除外し、精度低下を明示する。
- 計算ロジックはユーザー提供の参考サイト（四柱推命・陰陽五行側: suimei.starcrawler.net、算命学側: unkoi.com）の表示結果をゴールデンテストとして照合するが、これらのサイトへの実行時スクレイピングは行わない（独立実装＋事後照合のみ）。

### 5.4 分析生成（Claude APIによる解釈）
- 簡易分析・詳細分析の2モード（構成は本要件書末尾の付録を参照）。
- 事実・命式上の傾向・AIによる人事仮説・確認したいこと・提案を明確なラベルで分離する。
- MBTIは登録・表示するが主要分析ロジックには使用しない。
- 面談記録などのフリーテキストに含まれる指示文（プロンプトインジェクション）を実行しない。

### 5.5 データ更新
- 面談記録の追記は確認なしで保存（上書きせず日付ごとに追記）。
- 基本情報（生年月日・出生時間・出生地・性別・所属・役職等）の変更は保存前に必ず確認する。
- 削除は必ず確認し、原則として論理削除とする。
- すべての変更はアプリケーション側の検証・権限確認・監査ログ記録を経由する。LLMが直接シートを書き換えることはない。

### 5.6 変更履歴・取り消し
- すべての操作（登録・更新・削除・取り消し）を履歴として保存する。
- 「直前の変更を取り消して」等の自然言語指示による取り消しに対応する。
- 取り消し可能期限を設定し、期限超過後は明示的な再確認を要求する。

### 5.7 リッチメニュー
- 6項目: 人物を登録／簡易分析／詳細分析／相性分析／面談を記録／人材を比較。
- 相性分析・人材を比較はMVPでは「設計済み・第2段階で実装」の案内を返す。

### 5.8 データ保持
- 採用候補者: 選考終了から6か月後に削除（自動削除の実行は第3段階、MVPではフィールドとポリシーのみ実装）。
- 外部関係者: 手動削除。
- 社員: 退職後1年間保存。
- `retention_policy` / `retention_start_date` / `scheduled_delete_at` / `legal_hold` / `deletion_status` をMVPからデータモデルに含める。

## 6. 非機能要件

- 応答言語: 日本語。トーンは「経営者向け人事コンサルタント × 参謀」。占い師的な語り口を避ける。
- LINE Webhookは即座に200を返し、命式計算・Claude API呼び出しは非同期処理する。
- ローカル開発では外部キューなしでも動作すること（インメモリ/DBベースの疑似非同期）。
- 内部エラー・APIキー・スタックトレース・シート構造・DB情報をLINEへ表示しない。
- 監査ログ、論理削除、最小権限、依存関係脆弱性チェックを実施する。
- 日本国内出生者のみ対応（タイムゾーン `Asia/Tokyo` を既定とする）。

## 7. データ最小化方針

Claude APIには質問に必要な情報のみを送信する。健康・家族・退職相談等の機微情報は、明示的に必要な場合（退職・モチベーション分析等）のみ限定的に送信し、命式のみの質問や相性分析、採用可否判断には含めない。詳細は `docs/ai-prompt-design.md` を参照。

## 8. 付録: 回答モードの構成

**簡易分析**: 結論 / 本質 / 主な強み / 注意点 / 今の関わり方 / 命式上の主な根拠

**詳細分析**: 結論 / 命式による判断根拠 / 中心星・日干から見た本質 / 強み / 弱み・注意点 / 向いている役割 / 現在の大運・年運 / 必要に応じた月運 / 関わり方・伝え方 / 面談で聞く質問 / 人事上の提案 / 判断前に確認すべき事実 / 出生時間・データ不足などの注意

## 9. 対象外（MVP第1段階では実装しない）

相性分析・複数人比較・チーム編成・採用候補者専用分析ロジック・外部コンサルタント窓口選定・退職リスク自動仮説・自動削除実行・定期通知・海外出生者対応。データモデルとインターフェースは将来拡張できるよう準備する。

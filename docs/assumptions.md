# 仮定事項一覧

ユーザー確認なしに開発を進めるため、以下の妥当な仮定を置いた。変更が必要な場合はこのファイルを更新し、関連コード・ドキュメントも合わせて修正すること。

## 技術構成
- Pythonは3.12を採用（ローカル検証環境は3.10だったため、Dockerイメージでは`python:3.12-slim`を使用し、ローカルpipインストールもPython 3.10以上で動作するよう`requires-python = ">=3.10"`とした上でCI/本番は3.12に統一）。
- `line-bot-sdk` はv3系（`linebot.v3`）を採用。2026年6月時点でPython 3.10〜3.14対応。
- `ANTHROPIC_MODEL` の既定値は `claude-sonnet-4-6` 相当のプレースホルダーとし、実際のモデル名は石橋輝一のAnthropicアカウントで利用可能なものを`.env`で設定する前提（`docs/anthropic-setup.md`参照）。
- LINEの1メッセージ最大文字数は5000文字、1回のreply/pushで送れるメッセージ数は最大5件という現行仕様を前提に、設定ファイル（`app/config.py`の`LineLimits`）で管理し、仕様変更時に一箇所で調整できるようにした。

## 命式計算
- 計算基準日はデフォルトで `Asia/Tokyo`、日付変更は原則0時（`day_boundary_hour=0`）を初期設定とし、`CalculationPolicy`で23時基準にも切替可能にした。
- 年柱の切替基準は立春を採用（節切り）。月柱は二十四節気の節入りを使用する（旧暦の朔日基準は採用しない）。
- 真太陽時補正は初期設定でOFF（標準時のみ使用）とし、`CalculationPolicy.true_solar_time_correction`で有効化できるようにした。理由: 参考2サイトの表示が標準時ベースである可能性が高いという仮定。ゴールデンテストで不一致が出た場合は再検討する。
- 大運の順行・逆行は「年干の陰陽」×「性別」の組み合わせ（陽男陰女は順行、陰男陽女は逆行）という一般的な算命学・四柱推命の方式を採用。
- 大運開始年齢の計算方法は「生まれた日から次（または前）の節入りまでの日数 ÷ 3 = 年」という一般的な換算式（3日で1年、1日で4か月相当）を初期実装とした。
- 守護神の判定方法・中心星の算出方法・人体星図の配置方式は、公開されている一般的な算命学の計算方法（十干十二支の組み合わせから中心星を導く標準的なアルゴリズム）を実装した上で、ゴールデンテストにより検証待ちとした（`docs/calculation-policy.md`に詳細と検証状況を記載）。
- これらの初期設定は暫定であり、`docs/INPUT_REQUIRED.md`にある元サイトのスクリーンショット・出力結果が提供され次第、ゴールデンテストと照合し確定させる。

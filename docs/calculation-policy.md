# 計算ポリシー — 命式計算エンジン

最終更新: 2026-07-16。本ドキュメントはコード（`app/calculation/`）と対応する。ロジックを変更した場合は必ず本ドキュメントも更新すること。

## 1. 設計原則

- 命式・干支・大運・中心星等はすべて `app/calculation/` の決定論的Pythonコードで計算し、Claude（LLM）には計算結果の構造化データのみを渡す。LLMに推測させない。
- 四柱推命側と算命学側は別モジュール（`four_pillars.py` / `sanmeigaku.py`）とし、結果を統合はしても上書きはしない（`CalculationResult`が両方を別フィールドで保持）。
- 流派・サイトにより結果が変わりうる項目は `CalculationPolicy`（`app/calculation/policy.py`）で設定可能にし、コードにハードコードしない。

## 2. 検証状況の凡例

- **verified**: ユーザー提供のゴールデンテストデータと一致を確認済み
- **provisional**: 標準的とされるアルゴリズムで実装済みだが、ゴールデンテストでの照合待ち
- **unverified**: 流派差が大きく実装方針そのものが暫定。今後の精緻化が必要

## 3. 各項目の実装方式と検証状況

| 項目 | 実装方式 | 検証状況 |
|---|---|---|
| 年柱 | 1984年=甲子年を起点とする60干支サイクル。年柱切替は既定で立春基準（`YearPillarBoundary.RISSHUN`）、`JAN1`（西暦1月1日基準）にも切替可能 | provisional（1984=甲子は広く知られた事実。3件のサンプルで1991=辛未、1996=丙子、2002=壬午と、いずれも一般に知られる干支と一致することを確認済み） |
| 月柱 | 二十四節気のうち「節」（12個、立春=黄経315°から30°刻み）を太陽視黄経の低精度公式（Meeus）＋二分探索で計算し、節入り境界で月支を決定。月干は五虎遁（年干→寅月の天干対応表）で算出 | provisional（節入り日時はほぼ実測値と一致する精度（誤差分オーダー）だが、日付境界ぎりぎりのケースは要個別確認） |
| 日柱・日干 | 60干支の連続カウント。起点定数 `DAY_PILLAR_ANCHOR_DATE`（1900-01-01）と `DAY_PILLAR_ANCHOR_INDEX`（暫定値10）は未検証のプレースホルダー | **unverified — 校正必須**。`golden_tests/cli.py calibrate` で確定データ1件から起点を校正できる |
| 陰陽 | 天干・地支のインデックス偶数=陽、奇数=陰（標準） | provisional |
| 五行 | 天干・地支ごとの標準対応表 | provisional |
| 通変星（十神） | 日干と対象干の五行生剋関係＋陰陽の同異から算出する標準ロジック | provisional |
| 十二運 | 天干ごとの長生開始地支＋陽干順行・陰干逆行の標準表 | provisional |
| 蔵干 | 地支ごとの標準蔵干表（本気・中気・余気） | provisional |
| 五行バランス | 四柱（年月日、時柱があれば含む）の天干・地支の五行を単純カウント（蔵干は含めない簡易版） | provisional（蔵干を重み付けする流派もあり、将来オプション化を検討） |
| 時柱 | 出生時刻から2時間区切りの十二支を決定し、五鼠遁（日干→子時の天干対応表）で時干を算出。23時起点。出生時間未登録の場合は算出しない | provisional |
| 中心星 | 既定: 月支の蔵干（本気）と日干の通変星関係から十大主星名へ変換（`CenterStarMethod.MONTH_BRANCH_MAIN_QI`）。代替: 日支基準（`DAY_BRANCH_MAIN_QI`） | provisional |
| 十大主星 | 年干・月干・時干（日干は自分自身のため対象外）、および年支・月支・日支・時支の蔵干（本気）について、通変星と同一ロジックで算出し算命学の星名に変換 | provisional |
| 十二大従星 | 年支・月支・日支・時支について十二運と同一ロジックで算出し算命学の星名に変換 | provisional |
| 天中殺 | 日柱の60干支インデックスの十の位グループ（6グループ）から、そのグループで使われない地支2つを算出 | provisional |
| 守護神 | 簡易版: 五行バランスが最も少ない五行を「補うべき五行」として返す（`GuardianDeityMethod.BALANCE_WEAKEST_ELEMENT`） | **unverified**。伝統的な算命学の守護神判定（複数要素を組み合わせた高度な判定）は未実装 |
| 位相法 | 未実装（プレースホルダーの注記のみ） | **unverified** |
| 大運（四柱推命側） | 性別×年干陰陽で順行・逆行を判定（陽男陰女=順行、陰男陽女=逆行）。開始年齢は次/前の節入りまでの日数÷3年（`LuckStartAgeMethod.DAYS_DIV_3`）。以降10年ごとに月柱から干支を1つずつ進める/戻す | provisional |
| 大運（算命学側） | 未実装（`SanmeigakuResult.major_cycles`は空リストのまま） | 未実装 |
| 年運 | 対象年をその年の代表日時（7/1正午）で年柱と同じロジックにより算出 | provisional |
| 月運 | 対象日時について月柱と同じロジックにより算出 | provisional |
| 真太陽時補正 | 既定OFF。`CalculationPolicy.true_solar_time_correction`で将来的に経度差・均時差補正を追加できるようフラグのみ用意（計算ロジックは未実装） | 未実装 |
| 日付変更基準 | 既定0時。23時基準にも切替可能（`day_boundary_hour`） | provisional |

## 4. 精度に関する注意（自動的にユーザーへ表示する内容）

- 出生時間が未登録の場合、時柱を含まない旨と、一部の適性・運勢判断の精度が下がる旨を回答に含める（`FourPillarsResult.hour_pillar_omitted_reason`）。
- 日柱の起点定数が未検証であることは、`accuracy_notes`に含め、内部ログにも記録する。ゴールデンテストで確定するまでは、LINE回答内でも「日柱は検証中」である旨を明示する運用とする（`docs/ai-prompt-design.md`参照）。

## 5. ゴールデンテストによる校正手順

1. `docs/INPUT_REQUIRED.md` に記載の元サイト出力（画像・テキスト）を入手する。
2. `golden_tests/data/*.yaml` の該当項目に実測値を記入し、`status: verified` に更新する。
3. `python -m golden_tests.cli run` で差分を確認する。
4. 日柱がずれている場合は `python -m golden_tests.cli calibrate --name <person>` で起点定数の候補値を算出し、`app/calculation/four_pillars.py` の `DAY_PILLAR_ANCHOR_INDEX` を更新する。
5. 月柱・時柱等がずれている場合は、節入り計算やCalculationPolicyの設定（節切り方式、日付変更基準等）を見直す。

## 6. 参考基準サイトとの関係

- 四柱推命・陰陽五行側の参考: suimei.starcrawler.net の表示結果
- 算命学側の参考: unkoi.com の表示結果
